# Contribuir a Organigramix

Gracias por tu interés en contribuir.

## Cómo colaborar

1. Haz un fork del repositorio.
2. Crea una rama descriptiva:
   - `feat/nombre-cambio`
   - `fix/nombre-bug`
3. Implementa tu cambio con foco en claridad y mantenimiento.
4. Abre un Pull Request con contexto suficiente.

## Estándares mínimos

- Mantener compatibilidad con navegadores modernos.
- Evitar complejidad innecesaria.
- Preservar consistencia visual entre tema claro y oscuro.
- No mezclar refactor grande con bugfix sin explicación.

## Commits

Recomendado (no obligatorio):

- `feat: agregar ...`
- `fix: corregir ...`
- `docs: actualizar ...`
- `refactor: simplificar ...`

## Pull Requests

Incluye:

- Qué problema resuelve
- Qué cambió exactamente
- Cómo probarlo
- Riesgos o limitaciones

Si hay cambios de UI, adjunta capturas.

## Reporte de issues

Usa las plantillas de issue para bugs y mejoras.

## Código de conducta

Al participar en este proyecto aceptas [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
